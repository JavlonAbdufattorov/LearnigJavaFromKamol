# Java Code Convention (Код-конвенции Java)

Java Code Convention — это набор правил и рекомендаций, как писать Java-код **красиво, понятно, читаемо и профессионально**.
Эти правила используются в **Oracle**, крупных IT-компаниях и считаются стандартом для Java-разработчиков.

---

## 1. Именование (Naming Conventions)

### Пакеты (Packages)

Пакеты:
- пишутся **только маленькими буквами**
- обычно используют **обратный порядок доменного имени**

Пример:
package com.example.myapp;

Плохо:
MyPackage
com.Example.App

Хорошо:
com.company.project

---

### Классы и интерфейсы

Классы и интерфейсы пишутся в стиле **PascalCase** — каждое слово начинается с заглавной буквы.

Пример:
class UserService { }  
interface RunnableTask { }

Плохо:
user_service  
userservice  

Хорошо:
UserService  
RunnableTask

---

### Методы

Методы пишутся в стиле **camelCase**:
- первое слово с маленькой буквы
- имя обычно начинается с **глагола**

Примеры:
getName()  
calculateSum()  
printResult()

Плохо:
Get_name()  
Print_result()

---

### Переменные

Переменные тоже используют **camelCase**:
- имена должны быть **понятными и осмысленными**

Примеры:
int studentCount;  
double averageScore;

Плохо:
x  
a1  
cnt

---

### Константы (static final)

Константы:
- пишутся **заглавными буквами**
- слова разделяются подчёркиванием (**UPPER_SNAKE_CASE**)

Примеры:
static final int MAX_SIZE = 100;  
static final double PI = 3.14159;

---

## 2. Форматирование кода

### Отступы

- используется **4 пробела**
- табы не используются

Пример:
if (x > 0) {
    System.out.println(x);
}

---

### Фигурные скобки

- открывающая скобка `{` ставится **на той же строке**
- закрывающая — на новой

Правильно:
if (x > 0) {
    // code
}

Неправильно:
if (x > 0)
{
    // code
}

---

### Пустые строки

Пустые строки используются:
- между методами
- между логическими блоками кода

Пример:
int a = 5;
int b = 10;

int sum = a + b;

---

## 3. Комментарии

### Однострочные комментарии

Используются для коротких пояснений:
// This is a comment

---

### Многострочные комментарии

Используются для больших пояснений:
/*
Multi-line
comment
*/

---

### JavaDoc (очень важно)

JavaDoc используется для документации классов, методов и API.  
Из него автоматически генерируется документация.

Пример:
/**

Calculates the sum of two numbers.

@param a first number

@param b second number

@return sum of a and b
*/

public int sum(int a, int b) {

return a + b;

}

## 5. Хорошие практики (Best Practices)

- Пиши **понятный код**, а не короткий
- Один метод — **одна задача**
- Используй осмысленные имена
- Избегай **магических чисел**
- Не пиши всю логику в `main`
- Не используй `public`, если в этом нет необходимости
